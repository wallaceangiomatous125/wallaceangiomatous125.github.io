Open source

faucet-stream

Rust · Open source · Solo-authored

A production-grade, config-driven data-movement platform in Rust — with governance built in, that moves data one to two orders of magnitude faster than a Python data-integration runtime, as a single native binary or an embeddable typed library.

~96×faster than Meltano — 712k rows/s on a reproducible 1M-row move
~62×less memory — 11.8 MiB vs 724 MiB, with exact row parity
49connectors — 28 sources & 21 sinks, each an opt-in Cargo feature

It's a full runtime, not just connectors: incremental & resumable replication, change-data-capture, effectively-once delivery, upsert/delete write modes, data-quality checks, versioned data contracts, PII masking, freshness/volume SLA monitoring, dead-letter queues, adaptive batch sizing, secrets-manager interpolation, OpenLineage lineage, and built-in Prometheus metrics + tracing — all with zero per-connector code. One binary, many runtimes: run (one-shot), serve (an HTTP control plane with bounded concurrency, durable run history, event-driven triggers, and lease-based orphan fencing across a cluster), schedule (cron), and backfill (resumable historical replay) — and third parties extend it through a documented connector SDK and protocol.
